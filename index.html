Welcome to Code Club Auckland.

Here's our QBasic tutorial!

QBasic is an old programming language. It's not used much any more, but it used to be the most popular language for beginners.

Old computer magazines used to include programs written in Basic - not on a disk, but just printed in the magazine. You had to type all the code into your computer before you could run the program!

Installing QBasic

Windows:
1. Download the windows QBasic zip
2. Extract the zip file
3. Double-click on qb.bat

OS X:
1. Download the OS X QBasic zip
2. Extract the zip file
3. Double-click on qb.command

If you see a blue blocky screen then it's working!

If your mouse cursor is stick inside the QBasic screen, press Ctrl-F10 to release it.

Your first program

Write this

PRINT "Hello World"

Then press F5 (you might need to press Function + F5 on a laptop).

You should see a different screen. Ignore the blue stuff at the top, it's not from you - it's leftover from before.

At the bottom you should see Hello World and then Press any key to continue.

Asking a question

1 PRINT "What is your name?"
2 INPUT name$
3 PRINT "Hi, " + name$

INPUT keeps a record of what you typed in.

Making a choice

1 PRINT "What is your name?"
2 INPUT name$
3 IF (name$ = "Alice") THEN PRINT "Alice likes rabbits."
4 IF (name$ = "Bob") THEN PRINT "Bob likes burgers."

Jumping around

The computer goes through your program, line by line, as if it was a list of instructions.

1 PRINT "First it prints this"
2 PRINT "Then it prints this"

We can tell the computer to jump to a different line using GOTO.

Can you pretend to be a computer, and run this program in your brain? You need to keep track of what line number you are on.

1 PRINT "What's the password?"
2 INPUT pass$
3 IF (pass$ = "secret") THEN GOTO 10
4 PRINT "Wrong."
5 GOTO 1
10 PRINT "You win!"

Line numbers

If we put a line number at the start of a line, we can GOTO that line.

Numbers are hard to remember so most people use labels instead.

Let's remove all the line numbers from the code and make some labels, 'outside' and 'inside' instead.

outside:
PRINT "What's the password?"
INPUT pass$
IF (pass$ = "secret") THEN GOTO inside
PRINT "Wrong."
GOTO outside
inside:
PRINT "You win!"

The challenge

Make a game where you can move from room to room by choosing where to go.

Maybe one room has a password door, and the clue is in another room?

What have we learned?

The computer runs your code, one line at a time, as if it was following a recipe.

If you made any mistakes then the program might not work. Sometimes QBasic tries to tell you what is wrong and how to fix it. But it's not always good at explaining what's wrong!

You can tell the computer to jump or 'go to' other parts of the program.
